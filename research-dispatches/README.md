# research-dispatches/

**Formal open research questions** with hypothesis → prior constraints → findings → recommendation. One question per file.

**File naming:** `YYYY-MM-DD-<question-slug>.md`.

**Structure:**
1. Question + hypothesis
2. Prior constraints / what we already know
3. Findings (accumulated over time)
4. Recommendation
5. Re-opens appended as dated `## Update YYYY-MM-DD` sections — *not* rewrites

**Frontmatter required:** `type: research-dispatch`, `product:`, `status:` (raw → reviewed → incorporated), `priority:`.

**Distinct from:**
- `questions/` — which is pre-formal; a question moves here once we commit to researching it
- `experiments/` — which records *what happened when we tried X*; a dispatch may lead to an experiment

**Written on:** explicit ask to research a specific open question.
